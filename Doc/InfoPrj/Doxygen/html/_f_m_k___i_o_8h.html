This function is a callback function for dignostic/ error management.
.

Note
FMKIO module ask FMKCPU & FMKCDA modules if there is any error detected on respectively timer or channel and adc.
If an error is repertory and a callback function is given during FMKIO_cfg function, it will be called with debugging info from FMKCDA or FMKCPU module.

[in]f_debugInfo2_u8: Debug Info numero 2 UNUSED for now store information for each signals.

◆ FMKIO_Get_InFreqSigValue()

t_eReturnCode FMKIO_Get_InFreqSigValue ( t_eFMKIO_InFreqSig f_signal_e,
t_uint32 * f_value_pu32 )

Get the frequency input.
.

Note
Once the configuration is done, a function is called based on the meas trigger give in configuration function.
In the interrupt function we either increment a counter (count mode) or get the channel capture value and store it.
In this function we either get this value or the last saved capture value if an interruption is in progress and interruption function is writing in countainer "value".
If this happens retcode WARNING_BUSY is set and the calcul will be made on saved value.
Depending on configuration, this value is either a coutner, a period (ms) or a frequency (Hz)
Parameters
[in]f_signal_e: the input frequency signal, a value from t_eFMKIO_InFreqSig
[in]f_value_pu32: storage for value

Set an output in PWM configuration.
.

Note
Allow the user to choose among pwm intputs.
This function configure bspInit , call HAL_function, call FMKCPU to configure a timer in order to convert generate the pwm period and dutycycle.
IMPORTANT NOTE -> For SMT32, each signal has a timer and a channel based on confifguration. A timer has multiple channel that shared the frequency f_frequency_u32, once the timer set with this frequency the other signal (channel) will have the same frequency.
If the frequency is changed, it will be changed for every signal.
Parameters
[in]f_signal_e: the input analog signal, a value from t_eFMKIO_OutPwmSig
[in]f_pull_e: the input pull mode, value from t_eFMKIO_PullMode
[in]f_frequency_u32: the PWM frequency
[in]f_dutyCycle_u16: the dutycyle, value between 0 (0%) - 1000 (100%)
[in]f_puf_startNow_bll_e: wether or not the pulse generation should start immediately
[in]f_sigErr_cb: callbback function that will be called if an error occured
Return values
RC_OKRC_OK
RC_ERROR_PARAM_INVALIDRC_ERROR_PARAM_INVALID
RC_ERROR_ALREADY_CONFIGUREDRC_ERROR_ALREADY_CONFIGURED
